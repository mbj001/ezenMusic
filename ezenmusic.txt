5. 검색에서 급상승 키워드
급상승 키워드라는 테이블을 만들어서 순위를 관리해야 하는가?
급상승 키워드는 컬럼 2개로 키워드컬럼(1), 조회수컬럼(1) 정도로 생각중이긴한데
키워드에 들어갈 수 있는 것들은 아티스트이름, 앨범이름 정도로 하자


지금 검색기능은 아티스트, select 에서 like %str% 방식으로 서치하는 방식임
예를들어 "임영" 으로 검색을하면 임영이 포함된 아티스트, 앨범, 가사 들을 뽑아와서 각각 출력하는 방식임
즉,
아티스트 부분에서는 "임영"으로 아티스트 이름에 "임영" 이 있는지 axios.get => select 한번하고
곡 부분에서는 "임영"으로 아티스트 이름, 엘범이름, 노래이름에 "임영" 이 있는지  axios.get => select 한번하고
앨범 부분에서는 "임영"으로 아티스트 이름, 앨범이름에 "임영" 이 있는지 axios.get => select 한번하고
테마리스트 부분에서는 "임영"으로 테마리스트 이름에 "임영" 이 있는지 axios.get => select  한번하고
가사 부분에서는 "임영"으로 아티스트이름에 "임영"이 들어있는지 axios.get => select 한번하고
	=> 가사부분은 수정해야할 거 같은데 가사 내용 중에도 "임영" 이 들어있는지 판단해야할듯

이렇게 되면 검색 후 증가되어야 하는 키워드들이 문제임..
"임영"을 검색한다고 급상승키워드에 "임영" 이 들어갈 순 없잖음?

결국 
곡에서는 아티스트이름, 앨범이름, 노래이름
아티스트 부분에서는 아티스트 이름
앨범에서는 아티스트이름, 앨범이름
테마리스트 부분에서는 테마리스트 이름
가사부분에서는 아티스트 이름 	기준으로 검색을하니까

아티스트랑 테마리스트랑 가사 부분은 그냥 select 한 결과를 가지고 결과.length > 0 이면 
검색 테이블에서 조회수 + 1 해주면 됨

곡에서는 아티스트이름, 앨범이름, 노래이름을 where 로 찾는데 각각 중 어디에서 키워드를 찾아서 select 해온지를
